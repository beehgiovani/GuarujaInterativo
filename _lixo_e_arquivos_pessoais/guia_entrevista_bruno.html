FAQ de Alto Impacto (Modo Sênior)

🔴 P: "Como você lida com performance de grandes volumes no front?"
🟢 R: "No Web, uso On-demand Loading. No Android/Kotlin, uso Paging 3 e RecyclerView com DiffUtil. No Jetpack Compose, garanto a performance usando Stable types e LazyColumn, evitando recomposições desnecessárias através do State Hoisting."

🔴 P: "Qual sua estratégia para depurar e gerir logs em produção?"
🟢 R: "No Kotlin/Android, utilizo o BuildConfig.DEBUG para condicionar logs. Na camada de release, 'planto' o Timber com uma ReleaseTree que reporta apenas Provedores de Crash (Crashlytics) e uso o Proguard/R8 para remover chamadas de debug do código compilado."

🔴 P: "Hilt ou Koin? Como você escolhe a Injeção de Dependência?"
🟢 R: "Para projetos puramente Android, prefiro o Hilt por ser baseado no Dagger e gerar código em tempo de compilação (erros aparecem antes do app rodar). O Koin é excelente para KMP (Kotlin Multiplatform) por ser 100% Kotlin e não depender de geração de código (service locator), sendo mais leve em alguns contextos."

🔴 P: "Como você evita Memory Leaks ao usar Coroutines?"
🟢 R: "Sempre respeito o LifecycleScope. Nunca uso GlobalScope. No ViewModel, uso exclusivamente o viewModelScope, que cancela todas as tarefas automaticamente quando o componente é destruído, garantindo que não fiquem processos pendentes em segundo plano."

🔴 P: "Qual sua abordagem para manter a Consistência de Dados Offline?"
🟢 R: "Utilizo a abordagem Single Source of Truth via Room. O dado sempre vem do banco local; a API apenas atualiza o banco. No projeto Web (Guarugeo), usei lógica similar com IndexedDB no Service Worker para persistir os lotes e garantir funcionamento instável."

Backend & Cloud Stack

🌩️ Edge Functions (Supabase/Deno): "Uso para processar lógica próximo ao usuário, reduzindo latência. No meu projeto, elas gerenciam webhooks do Stripe/Pix e automações de enriquecimento de dados sem precisar de um servidor ligado 24/7."

🍃 Spring Boot vs Framework: "O Spring Framework é a base (Injeção de Dep., Bean Management). O Spring Boot é a camada de produtividade que traz 'autolayout' e um servidor embarcado, permitindo que eu foque na regra de negócio e não em configuração XML."

⚙️ Kafka (Mensageria): "Uso para arquiteturas distribuídas e alto volume. É imbatível para garantir que eventos não se percam (persistência no disco) e para desacoplar microserviços de forma assíncrona usando tópicos e partições."

☁️ AWS Lambda: "Trabalho com o conceito Serverless para tarefas curtas e disparadas por eventos (S3 upload, DynamoDB alteração). Foco em otimizar o 'Cold Start' e manter as funções granulares e independentes."

Engenharia de Performance & Estrutura

🏗️ Modularização: "Prefiro a modularização por Features em vez de apenas camadas. Isso permite que times diferentes trabalhem em recursos isolados, melhora o tempo de build (incremental) e facilita o reaproveitamento de código em outros apps ou módulos KMP."

💾 Queries Otimizadas: "No SQL/Room, foco em evitar o N+1 usando @Relation e @Transaction. No banco, não abro mão de Índices em colunas de busca frequente e sempre substituo o SELECT * por apenas os campos necessários, reduzindo o tráfego de memória."

📊 Collections (List, Set, Map):
• List: Coleção ordenada, aceita duplicatas. Uso para listas simples (ex: histórico).
• Set: Elementos únicos. Uso para garantir que não existam duplicatas (ex: IDs de lotes selecionados no mapa).
• Map: Pares Chave-Valor. Uso para buscas rápidas (O(1)) por ID ou Cache em memória."

🚀 Sealed Classes/Interfaces: "Uso para representar estados da UI de forma exaustiva (ex: UiState.Loading, UiState.Success, UiState.Error). Isso garante que o compilador me avise se eu esquecer de tratar algum estado no when."

🚀 Flows (State vs Shared): "Uso o StateFlow para estados que precisam de um valor inicial e 'replay' para novos observadores (como a lista de imóveis). Já o SharedFlow uso para eventos únicos que não devem ser repetidos, como navegação ou disparar um Toast."

🚀 Extensions & Functional: "Aproveito o poder das Extension Functions para limpar o código e funções de alta ordem como map, filter e flatMap para manipular coleções de dados de forma declarativa e concisa."

🚀 Null Safety: "Minha regra é: fugir do '!!'. Uso safe calls (?.), Elvis operator (?:) e let/run para garantir que o app seja resiliente a NullPointerExceptions, tratando nulos como estados esperados."
